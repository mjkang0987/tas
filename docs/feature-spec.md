# 기능명세서 — Take a Seat (TAS)

> 최종 수정: 2026-06-06

---

## 1. 제품 개요

헤어샵 전용 예약 관리 SaaS. 캘린더 기반 예약 관리, 고객 관리, 네이버 예약 동기화, 매출 분석을 단일 앱으로 제공한다.

### 1.1 역할 체계

| 역할 | 권한 범위 |
|------|----------|
| `owner` | 전체 설정, 팀원 관리, 초대 코드 발행 포함 모든 기능 |
| `manager` | 예약·고객·서비스·디자이너 CRUD, 매출 조회 |
| `staff` | 예약 생성·조회, 자기 담당 예약 관리 |

### 1.2 인증

- Google / Kakao / Naver OAuth 2.0
- NextAuth v5 JWT 전략 (세션에 `storeId`, `role`, `onboarded` 포함)
- 로그인 후 `onboarded: false` → `/onboarding` 강제 리다이렉트
- `onboarded: true` 상태에서 `/onboarding` 접근 → `/` 리다이렉트

## 2. 페이지별 기능

### 2.1 캘린더 (`/`)

메인 화면. 5가지 뷰 모드를 제공한다.

#### 2.1.1 뷰 모드

| 뷰 | 설명 |
|----|------|
| **주간** (Week) | 7일치 디자이너별 세로 타임슬롯 |
| **타임라인** (Timeline) | 가로 축 = 시간, 세로 축 = 날짜별 예약 바 |
| **월간** (Month) | 달력 형태, 날짜별 예약 도트 요약 |
| **연간** (Year) | 12개월 미니 캘린더, 예약 있는 날 강조 |
| **예약목록** (List) | 날짜 범위별 예약 목록 |

#### 2.1.2 예약 생성

1. 캘린더 빈 슬롯 클릭 → `ReservationCreate` 모달 오픈
2. 필수 입력: 고객, 서비스, 날짜, 시작/종료 시간
3. 선택 입력: 디자이너, 메모, 채널(현장방문/전화예약/네이버예약)
4. 저장 → `POST /api/reservations` → 스토어 즉시 반영

#### 2.1.3 예약 상세 / 수정

- 예약 클릭 → `ReservationDetail` 모달
- 수정 가능 항목: 서비스, 시간, 디자이너, 메모, 상태
- 상태 전환:
  - `active` → `completed` / `cancelled` / `noshow`
  - 취소 시 이유 입력 (선택)

#### 2.1.4 결제 처리

- 예약 상세 내 결제 레이어
- 결제 방법: 현금, 카드, 네이버페이, 지역화폐, 상품권, 적립금, 할인, 네이버 예약금 (복수 조합 가능)
- 결제 완료 → `paymentCompleted: true`, 포인트 자동 적립

#### 2.1.5 예약 이동

- 드래그 또는 이동 모달
- 디자이너 휴무일로 이동 시 확인 팝업

#### 2.1.6 충돌 배너

- 미해결 중복예약이 있을 때 캘린더 헤더 아래 표시
- "중복예약 N건이 있습니다 — 확인하기 →" 클릭 시 해당 충돌 해결 모달 오픈

### 2.2 주소록 (`/address`)

#### 기능 목록

| 기능 | 설명 |
|------|------|
| 고객 목록 | 이름·전화번호 검색, 정렬(이름/최근방문/포인트) |
| 고객 상세 | 예약 이력, 포인트 잔액, 메모 태그, 알러지/클레임/선호 노트 |
| 포인트 관리 | 수동 충전·차감, 결제 연동 사용/환급, 이력 조회 |
| 고객 병합 | 중복 고객 자동 감지 → 병합 제안 모달 → 확인 시 예약·포인트 이전 |

- 이름이 마스킹 패턴(`홍*동`, `*길*` 등)과 일치하는 고객이 있을 때 제안
- 이미 병합 검토한 고객 쌍은 localStorage에 기록하여 재노출 방지

### 2.3 설정 (`/settings`, `/settings/[tab]`)

#### 탭 구성

| 탭 | 기능 |
|----|------|
| **매출** (`revenue`) | 기간별 매출 KPI, 서비스/디자이너/채널별 차트, 일별 상세 |

- 조회 기간: 일간 / 주간 / 월간 / 연간
- KPI: 총 매출, 예약 건수, 신규 고객, 객단가
- 차트: 서비스별 / 디자이너별 / 채널별 비율

## 3. 네이버 예약 동기화

### 3.1 동기화 흐름

```
사용자 "동기화" 버튼 클릭
→ POST /api/naver-booking-sync
  → Google Gmail API로 네이버 예약 메일 파싱
  → 신규 예약 생성 / 수정 / 취소 처리
  → 충돌 감지 (시간 겹침)
→ calendarStore syncNotifications 업데이트
→ UI 알림 배지 + 배너 표시
```

### 3.2 알림 유형

| type | 설명 |
|------|------|
| `sync` | 신규 예약 동기화 완료 |
| `cancel` | 네이버 예약 취소 감지 |
| `conflict` | 기존 예약과 시간 충돌 |

### 3.3 충돌 해결 모달 (`NaverSyncConflictModal`)

- pending 충돌을 하나씩 순서대로 표시
- **확인**: `conflictStatus → 'confirmed'`, `read: true`, active pair 제거
- **나중에**: `conflictStatus → 'deferred'` (배너에서 계속 노출)
- 마운트 시 자동 확인: 더 이상 시간이 겹치지 않는 충돌 → 자동 `confirmed`

### 3.4 알림 노출 규칙

| 구분 | 노출 조건 |
|------|----------|
| 중복예약 | `conflictStatus !== 'confirmed'` 인 한 상시 노출 |
| 미확인 알람 | `read: false` + 생성 후 30일 이내 |
| 확인 알람 | `read: true` + 생성 후 7일 이내 |

### 3.5 배지 카운트 계산

```
badge = conflict(confirmed 아닌 건) + 일반알림(미읽음 + 30일 이내)
```

## 4. 알림 패널 / 전체 알림 모달

### 4.1 알림 패널 (빠른 확인)

- 헤더 벨 아이콘 클릭 → 드롭다운 패널
- 표시: 미해결 충돌 + 미읽음 일반 알림 (30일 이내)
- "모두 읽음" 버튼: 일반 미읽음 알림 일괄 처리 (충돌 제외)
- "전체 알림 →" 버튼: 전체 알림 모달 오픈

### 4.2 전체 알림 모달

3개 섹션으로 구분:

| 섹션 | 포함 항목 |
|------|----------|
| **중복예약** | `type === 'conflict'` AND (`confirmed 아님` OR `confirmed + 7일 이내`) |
| **미확인 알람** | `type !== 'conflict'` AND `read: false` AND 30일 이내 |
| **확인 알람** | `type !== 'conflict'` AND `read: true` AND 7일 이내 |

각 섹션 타이틀은 sticky (스크롤 시 고정), 섹션별 독립적으로 동작.

## 5. 포인트 시스템

### 5.1 적립 규칙

- 결제 완료 시 서비스 금액 × 적립률(%) 자동 적립 (`payment_earn`)
- 수동 추가 (`manual_add`), 수동 차감 (`manual_subtract`)
- 충전 (`recharge`): 기준금액 충전 시 보너스 추가

### 5.2 사용 규칙

- 결제 시 적립금 차감 (`payment_use`)
- 결제 취소 시 사용 포인트 환급 (`payment_adjust`)

### 5.3 충전 보너스 규칙

매장 설정에서 복수의 규칙 정의 가능:

```
예) 10만원 충전 시 1만원 보너스
    30만원 충전 시 5만원 보너스
```

## 6. 비즈니스 규칙

| 규칙 | 내용 |
|------|------|
| 예약 겹침 방지 | 동일 디자이너, 같은 시간대 중복 예약 시 경고 |
| 휴무일 예약 | 가능하나 확인 팝업 표시 |
| 퇴직 디자이너 | 신규 예약 배정 불가, 기존 예약은 유지 |
| 네이버 예약 취소 | 원본 예약 `status: 'cancelled'` 처리 |
| 포인트 최소 잔액 | 0 미만 불가 (차감 시 잔액 부족 경고) |
| 고객 병합 후 원본 | 병합된 고객 레코드 삭제 (병합 히스토리 보존) |
