# takeaseat (TAS) — 사용 가이드 (GUIDE)

> 미용실 예약·고객·매출 관리 서비스 **takeaseat**의 화면별 기능과 로그인 흐름을 정리한 이용 안내입니다.
> 코드/구조 관점의 인덱스는 [`index.md`](./index.md)를 참고하세요.

_최종 갱신: 2026-06-16 · 기준: `develop`_

---

## 0. 이용 방식 2가지

| 방식 | 설명 | 데이터 저장 위치 |
| --- | --- | --- |
| **로그인 사용** | Google/Kakao/Naver 소셜 로그인 후 매장 단위로 이용 | 서버(PostgreSQL) |
| **게스트 사용** | 로그인 없이 바로 체험. 대부분 기능 사용 가능 | 이 브라우저(localStorage) |

> 게스트로 쓰던 데이터는 나중에 로그인하면 **계정으로 이전(마이그레이션)** 할 수 있습니다.
> 단, 게스트 데이터는 해당 브라우저에만 있으므로 기기/브라우저를 바꾸면 보이지 않습니다.

---

## 1. 로그인 스텝

### 1-1. 소셜 로그인 (Google / Kakao / Naver)

1. `/login` 접속 → 사용할 소셜 버튼 선택.
2. (선택) **초대 코드**가 있으면 입력 후 진행 — 기존 매장에 멤버로 합류할 때 사용. 코드는 `tas-invite-code` 쿠키로 전달됩니다.
3. 소셜 인증 완료 → 서버가 계정을 생성/연결(`syncAuthUser`).
   - **초대 코드 없이 처음 로그인**: 새 매장이 자동 생성되고 본인이 **오너(owner)** 가 됩니다.
   - **초대 코드로 로그인**: 해당 매장에 **멤버(staff)** 로 합류합니다.
4. **약관 동의**가 필요하면 `/consent`로 이동 → 이용약관 + 개인정보처리방침 동의.
5. **온보딩(최초 설정)** 이 필요하면 `/onboarding`으로 이동(매장 미설정 시).
6. 완료되면 **메인 캘린더(`/`)** 로 진입.

### 1-2. 게스트로 시작

1. `/login` 하단의 **게스트 시작** 선택.
2. 약관 미동의 시 `/consent/onboarding/guest`로 이동 → 동의.
3. `/onboarding/guest`에서 매장명·서비스·디자이너 등 초기 설정(로컬 저장).
4. 메인 캘린더로 진입. 모든 데이터는 이 브라우저에 저장됩니다.

### 1-3. 로그아웃

- `/logout` 진입 → 로그아웃 버튼 → 세션 종료 후 `/login`으로 복귀.

### 로그인 관련 오류 안내

| 오류 | 의미 | 대처 |
| --- | --- | --- |
| `no-account` | 계정은 있으나 소속 매장이 없음 | 초대 코드로 재로그인하거나 새 매장 생성 |
| `OAuthAccountNotLinked` | 해당 소셜 계정이 다른 사용자에 연결됨 | 계정 연결/병합(설정 > SNS) 진행 |
| `sync-error` | 로그인 중 서버 동기화 실패 | 잠시 후 재시도 |
| 초대코드 오류 | `invalid-invite`/`invite-used`/`invite-expired` | 오너에게 유효한 코드 재발급 요청 |

## 2. 페이지별 기능

> 로그인 칸: **공개**(누구나) · **선택**(게스트 허용) · **필요**(로그인 필수) · **오너**(오너 전용)

### 2-1. 메인 캘린더 `/` — 선택
- 미용실 예약을 보고 관리하는 **핵심 화면**.
- **뷰 전환**: 일(`/day`) · 3일(`/three`) · 주(`/week`) · 월(`/month`) · 년(`/year`) — 모두 같은 캘린더로 연결.
- 예약 **생성/수정/취소/복구**, 예약 상세·결제 입력, 중복 예약 클러스터 표시.
- 상단에서 **디자이너 필터**·**고객 검색**, 시술 색상 범례 확인.

### 2-2. 고객 주소록 `/address` — 선택
- 고객 **검색**(이름/전화), 메모 **태그**(색상) 부여.
- 방문/취소/노쇼 통계, 충전(적립금) 관리.
- **고객 병합/병합 해제**(동명이인·중복 정리, 되돌리기 지원).

### 2-3. 설정 `/settings/[tab]` — 필요 · 오너
운영 설정 허브. 탭 구성:

| 탭 | 기능 |
| --- | --- |
| `revenue` | 매출 분석(디자이너별·기간별 KPI, Excel 내보내기) |
| `point` | 적립금 적립/차감·내역·정책 |
| `service` | 서비스(시술) 카탈로그·가격·소요시간 |
| `designer` | 디자이너/스태프 관리·근무 일정·색상 |
| `store` | 매장 정보·영업시간·휴무일 |
| `member` | 멤버(스태프) 관리 |
| `sns` | 소셜 계정 연결/해제·계정 병합 |
| `naver` | Gmail 연동 + 네이버 예약 동기화 |

> 멤버(staff)는 설정 접근이 제한되며 오너만 운영 설정을 변경할 수 있습니다.

### 2-4. 마이페이지 `/mypage` — 선택
- 닉네임 변경(중복 체크), 연결된 소셜 계정 확인, 로그아웃, **회원탈퇴**.
- 게스트의 경우 **로컬 저장 데이터 요약**(고객/예약/디자이너/서비스 수)과 초기화 버튼 제공.

### 2-5. 온보딩 `/onboarding`(·`/onboarding/guest`) — 선택
- 최초 설정 마법사(스텝 1~5): **매장명·업종 → 서비스 → 디자이너 → 포인트 → 검토/완료**.
- 로그인 사용자는 서버에 저장, 게스트는 로컬에 저장.

### 2-6. 문의 `/inquiry` — 선택
- **문의 작성**(이름·이메일·내용)과 **문의 내역** 조회. 로그인 시 이름/이메일 자동 입력.

### 2-7. 약관 동의 `/consent` — 선택(게이트)
- 이용약관 + 개인정보처리방침 동의 화면. 동의 후 원래 가려던 경로로 복귀.
- 약관 버전(`CURRENT_TERMS_VERSION`)이 갱신되면 전체 사용자에게 재동의를 요청합니다.

### 2-8. 약관·정책 문서 — 공개
- `/terms` 이용약관, `/privacy` 개인정보처리방침. (Aside 하단 링크에서도 접근)

## 4. 처음 시작하는 분을 위한 추천 순서

1. `/login`에서 소셜 로그인(또는 게스트 시작).
2. `/consent`에서 약관 동의.
3. `/onboarding`에서 매장·서비스·디자이너 설정.
4. `/`(메인 캘린더)에서 예약 등록 시작.
5. `/address`에서 고객 관리, `/settings`에서 매출·포인트·연동 설정.

## 5. 직접 실행해 보기

```bash
cd client
pnpm install
cp .env.example .env     # DB·OAuth 키 설정 (게스트 체험만 할 거면 생략 가능)
pnpm dev                 # http://localhost:3000
```

> DB·OAuth 키가 없으면 소셜 로그인은 동작하지 않지만 **게스트 모드로 전체 흐름 체험은 가능**합니다.
> 자세한 환경 변수는 `client/.env.example`을 참고하세요.
