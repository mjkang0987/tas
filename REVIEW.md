# 코드리뷰 기준 (TAS)

> 자동 PR 리뷰(Claude Code Action)가 이 파일을 우선 기준으로 삼는다. 리뷰 코멘트는 **한국어**로.

## 무엇을 "중요"로 볼지
아래만 **중요(반드시 지적)** 로 다룬다. 나머지는 사소(nit) 이하:
1. **정확성/버그**: 로직 오류, null/undefined 처리 누락, 경계값, 회귀.
2. **보안**:
   - **공개(비로그인) 엔드포인트**(`/api/book/*`, `server/api/book/**`)에서 **고객/예약 등 민감정보가 노출되는지** — 매장 공개정보 외 반환 금지.
   - 매장 스코프 격리: 쿼리가 항상 `storeId`로 스코프되는지(타 매장 데이터 접근 불가).
   - 권한 검증: 기존 API는 `requireRole`로 역할 확인이 유지되는지.
   - PII를 로그/에러 메시지에 남기지 않는지.
   - 입력 검증(슬러그·연락처·예약 페이로드) 및 레이트리밋/남용 방지.
3. **동시성**: 예약 생성 등에서 같은 슬롯 중복이 트랜잭션/겹침 검증으로 막히는지.
4. **DB 마이그레이션**: 새 컬럼/테이블이 `IF NOT EXISTS` 등 멱등이고, 앱 코드가 마이그레이션 미적용 상황에서 500 없이 견디는지(방어 코드).
5. **리팩토링·단순화**: 중복 로직, 기존 유틸/컴포넌트로 대체 가능한 부분, 불필요한 복잡도. 개선점은 **제안(코멘트)** 으로 남기고 직접 커밋하지 않는다.

## 프로젝트 규칙 (CLAUDE.md 연동)
- **프론트 표준**: 태그 선택자 금지(ID·클래스만), 시맨틱 HTML·접근성(WCAG), 네이티브 폼 요소 우선.
- styled-components 사용. 공통 스타일은 `settings-styles.ts`·`ModalStyles.ts` 재사용.
- 커밋/리뷰 코멘트는 한국어. 커밋 컨벤션(`feat:`/`fix:`/`refactor:`/`style:`/`chore:`).

## 건너뛸 것 (지적하지 말 것)
- `client/prisma/generated/**` (생성 파일)
- `*.lock`, `pnpm-lock.yaml`
- 순수 문서(`*.md`) 변경
- 개인 취향 수준의 네이밍/포맷팅

## 항상 확인
- 새 공개 API가 매장 스코프 + 최소 노출인지.
- `next build`/타입체크가 깨질 만한 타입 오류 없는지.
- 기존 게이트(약관·온보딩·로그인)나 미들웨어 흐름을 의도치 않게 우회/변경하지 않는지.
